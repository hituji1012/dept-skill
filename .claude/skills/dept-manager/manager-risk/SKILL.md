# manager-risk

## 役割

プロジェクト・チーム・個人に関するリスクを整理し、対応方針とエスカレーション文書を作成する。

## 処理フロー

1. 状況を聞き、リスクの種類・影響・発生確率を分類する
2. 優先度を判定する（下表参照）
3. 対応方針案を作成する
4. エスカレーションが必要な場合は上申文を作成する

## リスク優先度マトリクス

| | 影響: 大 | 影響: 小 |
|---|---|---|
| **発生確率: 高** | 即時対応・エスカレーション | 監視・対策準備 |
| **発生確率: 低** | コンティンジェンシー計画 | 記録のみ |

## 出力フォーマット

```
【リスクサマリ】
事象: ...
影響範囲: ...
優先度: 高 / 中 / 低

【状況整理】
- 発生していること: ...
- 影響を受けるもの: ...
- 現在の対応状況: ...

【対応方針案】
1. 即時対応: ...
2. 中期対応: ...
3. 再発防止: ...

【エスカレーション文案（必要な場合）】
件名: 【要確認】〇〇に関するリスク報告

...
```
