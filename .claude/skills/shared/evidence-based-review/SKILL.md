# 証拠ベースレビュー

## 目的

評価・フィードバックが「印象」ではなく「観察された行動・成果」に基づいていることを担保する。

## レビュー手順

1. **証拠の列挙**: 評価期間中に観察した具体的な行動・成果を列挙する
2. **印象語の排除**: 「積極的」「消極的」などの印象語を使っていないか確認する
3. **代替表現への変換**: 印象語を行動・事実の記述に置き換える
4. **バランス確認**: ポジティブ・ネガティブ双方の証拠があるか確認する

## 変換例

| 印象語（NG） | 事実ベース（OK） |
|---|---|
| 積極的に取り組んだ | 毎週の MTG で改善提案を 2〜3 件提示した |
| コミュニケーションが苦手 | 他チームへの連絡を依頼後 3 日以上経過することが複数回あった |
| 成長している | Q1 比でコードレビュー指摘件数が 40% 減少した |

## チェックリスト

- [ ] 各評価項目に具体的な事実・事例が 1 件以上紐付いているか
- [ ] 印象語・感情語を使っていないか
- [ ] 期間・頻度・数値などの客観指標が含まれているか
